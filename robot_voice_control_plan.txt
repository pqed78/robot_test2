[수정된 상세 계획서: ROS2 Humble & Isaac Sim 5.1 기반 음성 제어 로봇]

1. 시스템 환경 및 제약 조건
  - OS/Middleware: Ubuntu 22.04 + ROS2 Humble
  - Simulator: NVIDIA Isaac Sim 5.1
  - GPU: 24GB VRAM (Isaac Sim과 LLM 병행 구동 최적화 필요)
  - LLM 모델: Ollama 기반 Llama 3.1 8B (robot_commander 커스텀 모델)
  - Python: 3.10 (uv 기반 관리, CUDA 12.8 지원)

2. 시스템 아키텍처 (VRAM 효율적 설계)
  가. 음성 입력 (STT): Faster-Whisper (Large-v3 모델 사용)
      - compute_type="float16" 및 beam_size=5 설정을 통해 정확도 극대화.
  나. 지능형 파싱 (Ollama): 
      - robot_modelfile을 통해 정의된 Llama 3.1 8B 기반 'robot_commander' 사용.
      - 자연어를 JSON 액션(move_to, pick, place, stop)으로 변환.
  다. 사용자 확인 프로세스 (TTS): MeloTTS (로컬 고품질 음성)
      - GPU(CUDA) 가속을 통한 실시간 합성 및 aplay 기반 재생.
      - 로봇: "거실로 이동하여 오른팔로 컵을 잡을까요?" -> 사용자 음성 확인 대기.
  라. 로봇 제어 브릿지 (ROS2 Node):
      - Ollama의 JSON 출력을 분석하여 ROS2 Action(Nav2, MoveIt2)으로 전달 (진행 중).

3. Isaac Sim 5.1 연동 계획
  - Robot Model: URDF/Xacro 기반 양팔 차동 로봇 (Differential Drive + Dual Arm)
  - Controller: ROS2 Control (Diff Drive Controller, Joint Trajectory Controller)
  - Perception: Isaac Sim 내의 센서(Lidar, Camera) 데이터를 ROS2로 퍼블리시하여 실시간 네비게이션 및 매니퓰레이션 수행.

4. 세부 구현 단계 (업데이트됨)
  Step 1: Ollama 전용 모델 파일(Modelfile) 작성 (완료)
    - Qwen 2.5 7B 기반, JSON 출력 강제 및 영문 최적화 프롬프트 포함.
  Step 2: 음성 제어 통합 루프 개발 (완료)
    - robot_continuous_loop.py 구현: STT(Large-v3) -> LLM -> TTS(MeloTTS).
    - IDLE 및 CONFIRMING 상태 머신을 통한 사용자 확인 로직 적용.
  Step 3: Isaac Sim 5.1 로봇 시뮬레이션 환경 구축 (진행 중)
    - 차동 구동부(Nav2) 및 양팔(MoveIt2) 설정.
    - `isaac_ros_bridge`를 통한 실시간 명령 수신 확인.
  Step 4: ROS2-Ollama 액션 연동 (예정)
    - JSON 명령을 실제 ROS2 토픽/액션으로 변환하는 브릿지 완성.

5. 메모리 관리 및 최적화
  - 24GB VRAM 활용: Isaac Sim(10-12GB), LLM(5GB), STT/TTS(3GB) 할당 최적화.
  - Faster-Whisper의 float16 연산을 통해 GPU 메모리 점유율과 속도 균형 유지.
  - 실시간성 확보를 위해 MeloTTS의 speed=1.0 설정 및 aplay 비동기 호출 검토.
